Microfrontend Concept

Micro frontends are a pattern where web application UIs (front ends) are created from semi-autonomous sections that can be worked by various groups utilizing various technologies. Micro-frontend architectures resemble back-end designs where backends are created from semi-independent microservices. Each microfrontend has its own repository, its own package.json document, and its own build-tool configuration. Subsequently, each microfrontend has a free deployment process and an autonomous deploy/CI.

Single-SPA

single-spa is a framework for integrating different JavaScript microfrontends into a single frontend application. Architecting your frontend using single-spa offers several benefits, such as:

  • Use multiple frameworks seamlessly without page reloads (React, AngularJS, Angular, Ember, or whatever you're using)
  • Deploy your microfrontends independently
  • Compose code using a different framework without modifying your existing application
  • Lazy load code for improved initial load time
  • Examples: single-spa.org

Multi Page Application (MPA)

A Multi-page Application (MPA) is a web application composed of many pages that fully reload each time information changes. Any information change or update on the server triggers another page to be displayed in the application.

MPAs typically use traditional engineering practices. Each page sends a request to the server and completely refreshes all the content, even if only small parts of the information change. This can waste bandwidth and impact performance.

Module Federation

Module Federation is a module added to Webpack that provides flexibility and customization for building your application.

Module Federation allows a JavaScript application to dynamically load code from another application and share resources. If the consuming application lacks a dependency required by the federated code, Webpack will download the missing dependency from the federated source automatically.

Architecture of Repository Monorepo

A monorepo is a single repository that stores different projects. This approach is used by many large tech companies like Facebook, Google, Microsoft, Twitter, and Uber. Like a microfrontend, each project within a monorepo is independent and can be developed and maintained in isolation.

People unfamiliar with the concept often confuse a monorepo with a monolith. However, a monorepo is a code repository, while monoliths are typically large, single codebases. Monoliths can be used for both monoliths and microservices architectures.

Use tools that help, not because they're trendy.

Nx Workspace Tool + Monorepo (Perfect Union)

Nx Workspace is a TypeScript-based monorepo tool built on top of Angular DevKit. It provides a workspace, CLI, cloud-based build scheduling, and exceptional language-level IDE support.

Conclusion

This section explored several practices frequently used with microfrontends. The goal is not to definitively declare one approach as superior, but to provide a starting point for navigating your options.

Remember, 100% scalability is not always achievable. Always strive to select the approach that best fits the context and specific requirements of your project.
